üéØ Step-by-Step Setup: The Byron MN Blueprint

This is the exact process I used to set up the Byron MN Week 0 event. Follow these steps, and you'll have a professional-grade FMS running in under 2 hours.

‚è±Ô∏è Timeline: From Zero to Hero in 120 Minutes

Based on real deployments across Minnesota, this setup timeline assumes you have all hardware ready and one experienced mentor plus two students working together.

  1. Download FMS Freezy Arena
    Visit GitHub and search for "cpapplefamily cheesy-arena". Download the 2025-Reefscape-Week-0 release for your operating system. Extract to a dedicated folder on your FMS computer.
  2. Configure Network Hardware
    Set your router to 10.0.100.1 with DHCP range 10.0.100.20-199. This is non-negotiable - Driver Station software is hardcoded to look for FMS at 10.0.100.5.
  3. Position Alliance Switches
    Place one switch under the middle driver station of each alliance. Run 50-75ft CAT6 cables from each switch back to your scoring table router.
  4. Set FMS Computer IP
    Configure static IP: 10.0.100.5, subnet 255.255.255.0, gateway 10.0.100.1. This is the most critical step - get this wrong and nothing works.
  5. Connect Driver Stations
    Connect each team's driver station to their appropriate alliance switch using 15-20ft CAT6 cables. Set driver stations to DHCP (automatic).
  6. Launch and Configure FMS
    Start FMS Freezy Arena, access web interface at 10.0.100.5:8080. Create your event, add teams, configure match schedule, and test emergency systems.

üîß Troubleshooting: When Things Go Wrong

In three years of running FMS across Minnesota, I've seen every possible failure mode. Here's your survival guide for when (not if) things break.

üö® The "Oh Crap" Moment Survival Kit

"Nothing teaches troubleshooting like a gymnasium full of anxious teams staring at you while the FMS is down. Stay calm, think systematically, and remember - every problem has a solution." - Pranavagurubaran

üî¥ Driver Station Won't Connect

Symptoms: Red light, no communication

Solution Steps:

  1. Verify cable connection (physical layer)
  2. Check driver station IP (should be DHCP)
  3. Ping 10.0.100.5 from driver station
  4. Restart driver station software
  5. Check switch port LED indicators

üì° Robot WiFi Issues

Symptoms: Robot won't connect to field

Solution Steps:

  1. Verify robot radio configuration
  2. Check team SSID/password in FMS
  3. Test robot code network settings
  4. Reconfigure with FRC Radio Tool
  5. Check for WiFi interference

üéØ The Minnesota Troubleshooting Philosophy

Layer by Layer: Start with physical connections, then network layer, then application layer. Document Everything: What failed, when, and how you fixed it. Stay Calm: Panic is contagious, but so is confidence.
