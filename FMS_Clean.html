[ ] Configure all network hardware - [ ] Label all cables clearly - [ ] Create IP address documentation - [ ] Test with practice robot ### Setup Timeline (2 hours total) #### Hour 1: Infrastructure 1. **Download Software (15 min)** - Get latest Freezy Arena from GitHub - Extract to dedicated folder - Verify all files present 2. **Configure Router (20 min)** ``` IP: 10.0.100.1 Subnet: 255.255.255.0 DHCP: 10.0.100.20-199 Lease: 2 hours ``` 3. **Position Hardware (15 min)** - Router at scoring table - Switches under middle DS positions - Run CAT6 cables (50-75ft) 4. **Connect Components (10 min)** - Switches to router - FMS computer to router - Power everything on #### Hour 2: Configuration 5. **Set FMS Computer IP (10 min)** ``` IP: 10.0.100.5 (STATIC) Subnet: 255.255.255.0 Gateway: 10.0.100.1 DNS: 8.8.8.8, 8.8.4.4 ``` 6. **Launch FMS Software (20 min)** - Start Freezy Arena - Access at `10.0.100.5:8080` - Create event - Add teams 7. **Test All Stations (20 min)** - Connect DS laptops - Verify all show green - Test e-stop functions 8. **Final Validation (10 min)** - Run practice match - Test scoring - Verify displays ### üî• Byron Standard Validation - [ ] FMS responds at 10.0.100.5:8080 - [ ] All 6 driver stations connect - [ ] Test robot connects to WiFi - [ ] Emergency stops work - Change ALL default passwords 2. Disable WPS on WiFi 3. Use strong WPA2 keys (12+ characters) 4. Document credentials securely 5. Limit admin access ### Performance Optimization - Set WiFi channels to avoid interference - Configure QoS for FMS traffic priority - Monitor bandwidth during events - Keep spare equipment ready --- ## üîß Troubleshooting {#troubleshooting} ### Common Issues & Solutions #### üî¥ Driver Station Won't Connect **Symptoms:** Red communication light, no robot data **Solution Steps:** 1. Check physical cable connection 2. Verify DS is set to DHCP 3. Open command prompt, run: `ping 10.0.100.5` 4. If ping fails, check switch connection 5. Restart Driver Station software 6. Check switch port LED status #### üì° Robot Won't Connect to Field **Symptoms:** No robot communications, WiFi issues **Solution Steps:** 1. Verify robot radio configured for event 2. Check team number in FMS matches radio 3. Confirm SSID/password in FMS admin 4. Use FRC Radio Configuration Tool 5. Check for WiFi interference (use WiFi analyzer) 6. Try different WiFi channel #### ‚ö†Ô∏è FMS Software Crashes **Symptoms:** Web interface unresponsive, matches won't start **Solution Steps:** 1. Check FMS computer resources (CPU/RAM) 2. Restart FMS software 3. Clear browser cache 4. Check Windows Firewall settings 5. Verify no IP conflicts 6. Review FMS logs for errors ### Emergency Procedures **Complete System Failure:** 1. Switch to backup FMS computer 2. Use paper match schedules 3. Manual scoring if needed 4. Communicate with all teams 5. Document everything **Partial Failures:** - Single DS down: Move team to spare station - WiFi issues: Switch to backup router - Display problems: Use PA announcements --- ## üìö Resources & Support {#resources-support} ### Online Resources - [Chief Delphi FMS Forum](https://www.chiefdelphi.com/tag/fms) - [FMS Freezy Arena GitHub](https://github.com/cpapplefamily/cheesy-arena) - [Official FRC FMS Documentation](https://frc-docs.readthedocs.io/) - [FRC Discord #fms-help](https://discord.gg/frc) ### Minnesota Support Network - **RAYSE MN Mentors** - Local expertise - **Twin Cities FRC Alliance** - Regional support - **Rochester Area Teams** - SE Minnesota help - **Duluth Robotics Collective** - Northern support ### Emergency Contacts - **Author:** Pranavagurubaran S (Guru) - **Minnesota FTA Network:** Contact through FIRST - **Chief Delphi Emergency:** 24/7 community support ### Training Opportunities 1.
